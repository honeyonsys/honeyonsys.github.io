How to Host a Static Website on GitHub Pages

GitHub Pages is a free web hosting service offered by GitHub. It allows you to host static websites directly from your GitHub repositories. In this tutorial, we will show you how to host a static website on GitHub Pages and how to configure your domain settings to show the content on your GitHub Pages site.

Step 1: Create a GitHub Repository

The first step is to create a new repository on GitHub. This repository will contain your static website files. Follow these steps to create a new repository:

  1. Log in to your GitHub account
  2. Click on the “+” icon in the top right corner of the page and select “New repository”
  3. Enter a name for your repository and click “Create repository”

Step 2: Add Your Website Files to the Repository

Now that you have created a repository, you need to add your website files to it. Follow these steps to add your files:

  1. Clone your repository to your local machine using Git
  2. Add your website files to the repository
  3. Commit your changes and push them to GitHub

Step 3: Configure GitHub Pages Settings

After you have added your website files to your repository, you need to configure your GitHub Pages settings. Follow these steps:

  1. Go to your repository settings and click on the “Pages” tab
  2. Select the branch you want to use for your GitHub Pages site (typically the “main” branch)
  3. Choose the root folder for your GitHub Pages site (this should be the folder where your website files are located)
  4. Click “Save” to save your changes

Step 4: Configure Your Domain Settings

If you want to use a custom domain for your GitHub Pages site, you need to configure your domain settings. Follow these steps:

  1. Go to your domain registrar's website and log in to your account
  2. Create a new “A” record for your domain and point it to the IP address of GitHub Pages (185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153)
  3. Create a new “CNAME” record for your domain and point it to your GitHub Pages site (username.github.io)
  4. Save your changes

Step 5: Wait for DNS Propagation

After you have configured your domain settings, you need to wait for DNS propagation. This can take up to 24 hours. Once DNS propagation is complete, you should be able to access your website using your custom domain.
